
## Git / github regler
- Vi skriver ikke kode i databricks, vi skriver kode i Github
- Vi lager en ny branch der det gir mening, helst ved deomene som auth, test, experimantal, refactor etc.
- Vi commiter endringer heller flere ganger enn for få, det er viktig at vi har en pen git log
- Commit meldinger skal være på engelsk og korte. Holder med én setning. 
- Samme med PR meldinger, skal være på engelsk og korte, 1-3 setninger max. 
- ALDRI push
- ALDRI checkout
- ALDRI merge
- ALDRI sett deg selv som contributer eller co-author
